Vector - TODO

1. Add capacity_ field + reserve(n) + push_back(const T&) + push_back(T&&)
   - The single most important addition. Teaches amortized O(1) growth,
     the size vs capacity distinction, and reallocation strategy.
   - Must update ALL existing constructors, copy/move/swap, destructor
     to be capacity-aware.

2. emplace_back(Args&&... args)
   - Teaches perfect forwarding, variadic templates, and placement new.
   - Requires separating allocation (operator new) from construction
     (placement new), which is how real allocators work.

3. begin() / end() / cbegin() / cend()
   - Raw pointer iterators. Unlocks range-for and all STL algorithms.
   - Trivial to implement but huge payoff — your Vector becomes
     compatible with std::sort, std::find, std::copy, etc.

4. clear() + empty() + data()
   - Small but essential API. clear() sets size_=0 without deallocating,
     empty() is idiomatic, data() gives raw access.

5. Fix operator>> memory leak
   - If new throws during buffer resize, the old buffer leaks.
   - Wrap buffer in std::unique_ptr<U[]> for RAII safety.
